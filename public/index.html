Developer API REST

Generate on-brand announcement copy programmatically — useful for integrating into your CRM, automations, or internal tools. Two endpoints: /api/announce does research + generation in one shot, /api/refine skips research and iterates on existing copy.

Password: the name of our AI Slack channel. Reach out to Daniel if you need access.

POST /api/announce Research brand voice + generate copy in one call (~60–90s)
Request
# Paste this into your terminal curl -X POST https://tone-dashboard.vercel.app/api/announce \ -H "Content-Type: application/json" \ -H "x-api-key: <password>" \ -d '{ "name": "Rubirosa", // required "website": "https://...", // required "announcement": "A pizza drop", // required "types": ["sms","email"], // required — see valid values below "exampleContent": "..." // optional }'
Response
{ "brand": { "name": "...", "website": "..." }, "tone": { "adjectives": [...], "summary": "..." }, // pass to /api/refine "sources": [{ "label": "...", "url": "..." }], // pass to /api/refine // whichever formats you requested: "sms": "...", "emailSubject": "...", "emailBody": "...", "instagram": "...", "bannerHeadline": "...", "bannerSubhead": "...", "push": "...", "newsletter": "..." }
POST /api/refine Iterate on existing copy — skips research, just rewrites (~10–20s)
Request
# Pass back tone + sources from /api/announce, add an iteration note curl -X POST https://tone-dashboard.vercel.app/api/refine \ -H "Content-Type: application/json" \ -H "x-api-key: <password>" \ -d '{ "name": "Rubirosa", // required "website": "https://...", // required "announcement": "A pizza drop", // required "types": ["sms","email"], // required "tone": <tone from /api/announce>, // required "sources": <sources from /api/announce>,// required "iteration": "make the SMS more urgent" // required — what to change }'
Response
{ // only the copy fields — no tone or sources // pass tone + sources through yourself for further iterations "sms": "...", "emailSubject": "...", "emailBody": "...", "instagram": "...", "bannerHeadline": "...", "bannerSubhead": "...", "push": "...", "newsletter": "..." }
types — valid values
sms160-char SMS in brand voice
emailSubject + body, 2–4 paragraphs
instagramCaption, 2–5 lines
bannerHeadline (8 words) + subhead (20 words)
pushPush notification, max 100 chars
newsletterNewsletter blurb, 2–3 paragraphs
Notes
Latency/api/announce: 60–90s (research + generation). /api/refine: ~10–20s (generation only)
TimeoutSet your client timeout to at least 120s for /api/announce
Auth errors401 means wrong or missing x-api-key header
